Spring Framework

IoC

IoC(Inversion of Control): 設計原則，取代new物件的依賴
DI(Dependency Injection): 用Annotation實現IoC

讓物件的產生與配置委託給IOC容器(這裡說的也就是Spring IOC Container)，讓程式到了需要時，讓容器來幫我們注入

Object obj = new Object();
改成
Object obj = ApplicationContext.getBean(Object.class);

AOP代理模式，不動原有設計加新功能

AOP(Aspect Oriented Programming with Spring): 一種概念與一系列標準
Proxy Design(代理模式): 預測使用者行為，自動裝配

  1. 靜態代理(Static Proxy): 程式還沒運行前就確立了整個物件的包裝與邏輯
  2. 動態代理(Dynamic Proxy): 代理物件交由程式運行時生成

Hibernate

配對應用程式模型物件和數據庫表格的資料的框架

JPA(Java Persistence API)

通過註解或XML描述, 來指定對象與關係表之間的映射關係(Mapping), 並將實體對象持久化到資料庫，只是interace，Hibernate就是實現了JPA接口的Object-relational mapping (ORM)框架

Spring Data JPA

簡化JPA開發的框架，不implemention接口的情況下, 直接對資料庫做訪問與操作

時程: 2023/2/1- 2023/3/1

注意事項與改進措施

  1. Annotation取代實作interface
  2. POJO

Spring的設計模式。

IoC(Inversion of Control): 設計原則，取代new物件的依賴

Java Message Service（JMS）

Client-Server訊息的傳送、接收、頻道（Channel）、主題（Topic）、佇列 （Queue）等標準介面。

AOP代理模式，不動原有設計加新功能

AOP(Aspect Oriented Programming with Spring): 一種概念與一系列標準
Proxy Design(代理模式): 預測使用者行為，自動裝配

  1. 靜態代理(Static Proxy): 程式還沒運行前就確立了整個物件的包裝與邏輯
  2. 動態代理(Dynamic Proxy): 代理物件交由程式運行時生成

JPA(Java Persistence API)

通過註解或XML描述, 來指定對象與關係表之間的映射關係(Mapping), 並將實體對象持久化到資料庫，只是interace，Hibernate就是實現了JPA接口的Object-relational mapping (ORM)框架

Spring Data JPA

簡化JPA開發的框架，不implemention接口的情況下, 直接對資料庫做CRUD，底層使用了Hibernate

Hibernate

配對應用程式模型物件和數據庫表格的資料的框架

Data Access Object (DAO)

封裝資料庫進行持久層的操作

Spring MVC(Model.View.Controller)

  1. View: Postman( Get, Post, Put, Delete) in Thymeleaf/JSP or command
  2. Controller: API Layer 以前是Struts負責，溝通後端Servlets( Model)與前端JSP( View)
  3. Model: Service(Business Logic)Spring IoC，管理Strut和 Hibernate
  4. DAO: Presistence Logic Hibernate封裝SQL
  5. DB 執行SQL語句
  • SSH(Spring + Struts + Hibernate)

注意事項與改進措施

  1. Annotation取代實作interface
  2. POJO

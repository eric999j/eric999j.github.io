與Spring一樣採用MVC架構

Presentation Layer

  • Intercepting Filter Pattern - 分類與轉送不同需求到適當元件, 用於認證、日誌記錄等。
  • Front Controller Pattern - 所有請求的單一處理程序。用於中心化的請求處理/路由。
  • View Helper Pattern - 封裝視圖渲染邏輯。提高重用性和分離關注點。
  • Composite View Pattern - 從多個源中合併數據到一個視圖中。

JavaFX v.s. PrimeFaces: JavaFX 是swing 後繼, 內建Ajax; PrimeFaces基於JSF建構企業網頁UI

  • PrimeFaces
    面向Java EE的UI庫,可以想成java版的jQuery, 但PrimeFaces在伺服器運行, 通過JSF生成HTML

  • JSF(JavaServer Faces): 表示層
    Java EE的MVC web框架,定義標準UI與提供API, 用Ajax實現無頁面刷新, 良好支援Spring, Hibernate之類的其他標準,用”action”呼叫controller, p:commandButton的update跳轉到下個xhtml的p:pannelGroup id

    • core: JSF 驗證與轉換Tag
    • html: HTML Tag
    • facelets: 頁面區塊

使用
main/resources/A.properties
app.title=標題

main/webapp/jsff/B.xhtml
’#{A[‘app.title’]}‘

JSF執行流程

Six Stages of JSF life cycle

  1. Restore Phase of view
  2. Process events: Apply step of request values
  3. Process events: Step of process validation
  4. Process events: Step update of model values
  5. IProcess events: Invoke step of application
  6. Render response step

  • JSP(JavaServer Pages)
    靜態網頁渲染, 不像JSF要求MVC, 可以隨意加入Java程式, 與JSF可合併使用, 或說JSP可以被包含在JSF

  • XHTML: 繼承HTML並融合XML的嚴謹

Business Layer

  • Business Delegate Pattern - 處理分層的互動。解耦表示層和業務層。
  • Service Locator Pattern - 根據需要傳回服務實例。
  • Session Facade Pattern - 提供業務服務的簡化和統一視圖。屏蔽客户端的複雜性。
  • Transfer Object Pattern - 在層間傳輸的聚合數據。減少遠程調用。

JSF Scope完全符合CDI, Jakarta Scope更加擴展 (例: @ViewScoped - 當前畫面存活，比CDI的@RequestScoped更長)

  • 規範
    EJB(Enterpise JavaBean): 一種伺服器規範, 在JavaEE規範下完成訪問後端資料庫與安全, 部屬等事務

  • 作法
    CDI(Contexts and Dependency Injection): Java EE6開始引入的依賴注入框架, EJB3.0開始用CDI實現依賴注入

  • 實現
    Payara: 支持EJB的開源應用

Spring只實現部份Java EE, JSF還沒辦法替代

Persistence Layer/Integration Layer

  • Data Access Object Pattern - 持久化使用抽象API, 隔離應用, 業務, 持久層。
  • Web Service Broker Pattern - 使用XML, web協定讓不同服務交流。

JPA v.s. JPL: 都是ORM架構, JPA有整個Java生態系, 但發展放緩; JPL持續發展, 但較不成熟

  • JNDI(Java Naming and Directory Interface)
    為應用中對象命名,EJB透過它查找對象

  • JPA(Java Persistence API): 業務與資料層
    實現ORM的框架,在Java EE有自帶JPA外,開源的EclipseLink也是種實現, Spring 中的實現是Hibernate

  • JTA(Java Transaction API)
    Java管理事務的規範, Java EE自帶JTA, 方便管理事務

提示技術演變

  • 零樣本(Zero-shot Prompting): 不給問題範例直接問[複雜模型?]
  • 少量樣本（Few-shot Prompting）: 只給少量樣本[進階推理?]
  • 鏈式思考(Chain-of-Thought Prompting, COT): 添加”逐步思考”提示
    • CoT 技術局限
      • 自我糾正能力不夠完善
      • LLM 容易受到無關上下文的干擾
      • 推理過程會受到前提順序的影響
  • 思維樹(Tree of Thoughts，ToT): 引導模型使用樹狀結構演算法分析
  • 搜尋增強生成(Retrieval Augmented Generation，RAG)
    • 透過強化學習方式擷取, 搜尋,推理, 生成
    • 既有知識結合即時資訊(有搜尋避免幻覺)
  • 自動推理並使用工具(Automatic Reasoning and Tool-use, ART)
    • 不須人類介入, 自動搜索推論
  • 自動提示工程(Automatic Prompt Engineering, APE)
    • 添加”逐步解題”提示
  • ReAct
    • 上下文感知, 推理, 決策, 模擬人類學習過程
  • Reflexion(強化學習, GAN and more)
    • 參與者（Actor）,評估者（Evaluator）,自我反思（Self-Reflection）
    • 反思提高下一步決策

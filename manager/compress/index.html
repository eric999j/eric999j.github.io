封存格式 (Archive format)

文件打包的容器

tar (.tar) 系列本身僅打包，不壓縮

* tar (GNU): GNU tar 版本，功能完整。
* tar (POSIX pax 交換) / 受限制的 POSIX pax 交換: 支援 POSIX 標準的檔案屬性與交換資訊，更兼容不同系統。
* tar (POSIX ustar): 舊版 POSIX 格式，限制檔名長度與檔案大小，兼容性極高。

壓縮方法 (Compression Method)

決定壓縮率、速度與資源佔用

xz (.xz)

* 壓縮率極高，尤其對文字檔特別有效。
* 壓縮比 bzip2 還高，但壓縮速度慢。
* 常見於 Linux 發行套件（例如 .tar.xz）。

Zstandard (zstd)

* 由 Facebook 開發，壓縮/解壓極快。
* 支援可調壓縮等級（在壓縮率與速度間靈活折衷）。
* 適合遊戲資源、即時應用、雲端備份。

BZip2 (.bz2)

* 壓縮率高於 gzip，但速度慢（壓縮特別慢）。
* 適合大檔案與壓縮率優先的需求。
* 單線程壓縮，不適合超大檔案的快速處理。

7zip (.7z)

* 壓縮率非常高（特別是大量重複數據）。
* 支援強大的 AES-256 加密。
* 跨平台支援較少（Windows 原生不支援，需要額外軟體）。
* 適合需要高度壓縮與安全性的資料備份。

Gzip (.gz)

* 壓縮率適中，壓縮/解壓速度非常快。
* Unix/Linux 常用，適合網路傳輸（HTTP 壓縮）。
* 適合 CPU 資源有限但速度要求高的情況。

ZIP 壓縮檔 (.zip)

* 跨平台相容性好，Windows/macOS/Linux 原生支援。
* 壓縮率較 7z 低，但壓縮/解壓速度快。
* 支援基本加密（AES 或 ZipCrypto）。
* 適合日常資料傳輸或跨系統使用。
